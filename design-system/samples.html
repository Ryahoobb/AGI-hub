Pattern 2 / Broadsheet — クリーム背景ロングリード（accent: deepblue）
Longread / Geopolitics

知能の国境線

AGI開発が国家戦略の中心に移動するとき、研究の倫理は誰が管轄するのか。

国境。PHOTOGRAPH: UNSPLASH

2023年以降、AGI研究は確実に地政学の一部になった。OpenAI、Anthropic、DeepMindといった民間研究所が持つ計算資源は、十年前の国家プロジェクトを凌駕する規模に達し、そして同時に、その影響は特定の国境の内側で完結しなくなった。

この移行は静かだったが、後戻りできない。中国の国家AI計画はすでに7年を経過し、EUのAI法は民間モデルの規制を制度化した。アメリカは輸出規制という形で半導体サプライチェーンを知能生産の関門として再定義した。

I

計算資源という国境

NVIDIAのH100チップの生産能力がそのまま各国のAI研究能力の上限になった時点で、知能の地理が書き換えられた。問題は誰がどれだけの計算を持つかではなく、誰がその流通を統制するかに移った。

計算は新しい石油ではない。計算は新しい国境である。 — 本文より

この比喩は誇張ではない。石油と違い、計算は物理的な産地を持たず、しかし同じ重みで国家の生存戦略を規定する。

Image Placement Gallery — 10 variations of image size × layout
Reference / Image Placement

画像配置10種

本文幅680pxを基準に、サイズと配置の組み合わせで10通りのバリエーションを定義する。記事で使うときは3種類以内に抑え、視覚的休符としての価値を保つ。

1. Full-bleed ― 画面端まで達する全幅

セクションの切れ目、特集オープナー、視覚的休符として使う。1記事に1回まで。

2. Wide ― 本文幅を超えるが画面端までは達しない

主要ヒーロー、引きのある資料画像。本文の流れを保ちつつ視覚的な広がりを出したいとき。

3. Content-width ― 本文と同幅

標準配置。説明的な図版、引用画像、本文の流れを止めずに挟む画像に使う。

4. Center small ― 本文より小さく中央

補助図、小さめのポートレート、概念図。本文幅を使い切ると重すぎる場合に選ぶ。

5. Float left ― 左寄せ・本文回り込み

画像の右側に本文が回り込む配置。人物紹介、小カット、短い説明に寄り添う図版として機能する。段落は最低3段落以上続けて、floatを十分に消費する必要がある。

1段落だけで終わるとレイアウトが崩れるため、事前に段落の長さを見積もっておく。モバイル表示では自動で落ちることを前提にして、本文の論理が画像なしでも通るように書く。

この手法は長年の編集デザインの定番だが、デジタル環境では忘れられがちで、WIREDも一部の特集記事でしか使っていない。だからこそ使うと紙面感が出る。

6. Float right ― 右寄せ・本文回り込み

画像の左側に本文が回り込む配置。サイドノート、引用画像、補助資料としての画像を本文の流れに差し込むときに使う。右寄せの方が視線の流れを妨げにくいため、長文中では左寄せより頻度が高くなる。

Broadsheet（ロングリード）パターンと相性がよい。セクション冒頭の引き画像として置くと、本文の密度を保ちながら視覚的な変化を加えられる。

floatを終えた次の見出しは clear: both を効かせる。CSS側で h2 に clear: both を付けておくと事故が減る。

7. Side by side ― 2枚並列

Before/After、対比、二項図。幅は本文幅内で2分割される。

8. Grid 3 / Grid 4 ― フォトエッセイ用

3枚または4枚の均等グリッド。フォトエッセイ、研究拠点群、作例一覧に使う。

9. Overlay ― 画像の上にテキスト重ね

特集カバー、パート扉。画像をグレースケール + 暗くし、白文字を重ねる。1記事に1回まで。

10. Inline ― 段落内の小画像

このロジックは の初期版で既に見られた。アイコン、ロゴ、絵文字代替として使う。

段落先頭にサムネイルを配置し、本文が右側に回り込む短い紹介ブロック。紹介カード、関連記事ブロック、用語定義の先頭画像として機能する。サムネは96px固定で、段落の高さを自然に消費するよう長めに書く。

Chart System — モノトーン + アクセント1色のデータビズ
Chart 01 / Vertical Bar

計算コストの推移

時系列比較の基本形。直近のデータ点だけをアクセント色で強調し、それ以外はモノトーン階調で抑える。凡例は使わず、直近バーの上に数値を直接書く。

Chart 02 / Horizontal Ranking

ランキング・構成比の表示

カテゴリが6項目以上ある、またはラベルが長い場合は横棒を選ぶ。棒の末尾に数値を直接書き、凡例と軸ラベルを省く。注目カテゴリだけアクセント色に切り替える。

Chart 03 / Line

複数系列の時系列比較

時系列で2〜4系列を比較する定番。凡例は使わず、線の末端に直接ラベルを置く。主役の1系列だけアクセント色にし、他系列はグレー階調で抑える。

Chart 04 / Area + Annotation

累積成長とアノテーション

エリアチャートは累積量や構成比の推移を示す。特定の時点に注釈を添えることで、データに物語を重ねる。FT/Economist流のドロップインアノテーション。

Chart 05 / Slope

2時点比較（スロープチャート）

2時点の変化を複数カテゴリで比較するならスロープが最も読みやすい。大きく変化した項目だけをアクセント、他はグレーで重ねる。Tufteが推奨する最も情報密度の高い形式の一つ。

Chart 06 / Dumbbell

差分の強調（ダンベル）

同一カテゴリの2時点または2条件の差分を強調したいときに使う。バーの代わりに2点とそれを結ぶ線で表現するため、ギャップが視覚的に読み取りやすい。

Chart 07 / Scatter

相関・分布の可視化

散布図は2変数の相関を示す。外れ値や注目データ点だけをアクセント色にして、全体分布はグレーの点群で表現する。軸ラベルは左下に控えめに配置。

Chart 08 / Stacked Bar

構成比の推移

累積構成比。階調で複数カテゴリを重ねる。モノトーン階調3段階+アクセントの組み合わせで、最大4カテゴリまで扱える。それ以上は可読性が落ちるため使わない。

Chart 09 / Big Number

数値1点を際立たせる

グラフ化するほどデータ系列がない場合、巨大な数字を1-4個並べる。WIRED/FT短報やKPIダッシュボードで多用される。軸も線もいらない、ただ数値をタイポグラフィとして扱う。
